    **1. Create `worker/src/routes/chat.ts`** — Hono route group for buddy chat:

    Export a Hono instance `chatRoutes` with these endpoints (all require authGuard + sanitize):

    a) **POST /message** — Send a chat message
       - Extract `{ message, conversationId }` from request body
       - Get userId from `c.get("userId")`
       - If no conversationId, generate one: `crypto.randomUUID()`
       - Persist user message to `chatMessages` table
       - Call `detectRoadmapIntent(message)`
       - **If roadmap intent detected:**
         - Extract topic via `extractTopicFromMessage(message)`
         - Trigger Workflow: `const instance = await c.env.CONTENT_WORKFLOW.create({ params: { topic, userId, conversationId } })`
         - Persist assistant message with type "generation_started": `JSON.stringify({ type: "generation_started", workflowRunId: instance.id, topic })`
         - Return `c.json({ type: "generation_started", workflowRunId: instance.id, topic }, 202)`
       - **If conversational:**
         - Fetch last 20 messages for this conversation from D1 (ORDER BY createdAt DESC, LIMIT 20, then reverse)
         - Build messages via `buildChatMessages(history, message)`
         - Stream: `const stream = await c.env.AI.run("@cf/meta/llama-3.3-70b-instruct-fp8-fast", { messages, stream: true })`
         - Return SSE response: `new Response(stream as ReadableStream, { headers: { "Content-Type": "text/event-stream", "Cache-Control": "no-cache" } })`
         - NOTE: Do NOT add `response_format` here — streaming is incompatible with structured output (Pitfall 2)

    b) **GET /conversations** — List user's conversations
       - Fetch distinct conversationIds with latest message timestamp for current userId
       - Return array of `{ conversationId, lastMessageAt, preview }` (preview = first 100 chars of last message)

    c) **GET /conversations/:conversationId/messages** — Get messages for a conversation
       - Verify conversation belongs to user (IDOR check via userId match)
       - Return messages ordered by createdAt ASC

    d) **GET /status/:workflowRunId** — Poll workflow generation status
       - Fetch roadmap from D1 where workflowRunId matches
       - Return `{ status: roadmap.status, roadmapId: roadmap.id }` (status is "generating" | "complete" | "failed")
       - If no roadmap found with that runId, return 404

    **2. Create `worker/src/routes/roadmaps.ts`** — Hono route group for roadmaps + lessons:

    Export a Hono instance `roadmapRoutes` with these endpoints (all require authGuard):

    a) **GET /** — List user's roadmaps
       - Query roadmaps where userId = current user, ORDER BY createdAt DESC
       - For each roadmap: count total lessons and completed lessons (LEFT JOIN lessonCompletions)
       - Return `{ id, title, topic, complexity, status, totalLessons, completedLessons, createdAt }[]`

    b) **GET /:id** — Get roadmap detail with nodes and completion state
       - Verify ownership: roadmap.userId === current user (IDOR prevention per SEC-03 contract)
       - Return roadmap with parsed nodesJson + list of completed lessonIds for this user
       - This powers the node tree UI — frontend computes locked/available/complete states (per Pitfall 7)

    c) **GET /:id/lessons/:lessonId** — Get lesson content with quiz (WITHOUT answer key)
       - Verify lesson belongs to a roadmap owned by current user
       - Return lesson content + quiz questions
       - **CRITICAL SECURITY (Pitfall 6, UX-02):** Quiz questions MUST include `questionText`, `questionType`, `optionsJson`, `order` but MUST NEVER include `correctOptionId` or `explanation` before answer submission
       - Strip `correctOptionId` and `explanation` from the response

    d) **POST /:id/lessons/:lessonId/complete** — Mark lesson as complete
       - Verify ownership
       - Insert into `lessonCompletions` (idempotent — check if already exists)
       - Return `{ completed: true }`

    e) **POST /quiz/:questionId/answer** — Submit quiz answer
       - Accept `{ selectedOptionId }` in body
       - Fetch question from D1 (verify quiz belongs to user's roadmap via join)
       - Compare selectedOptionId with correctOptionId
       - Return `{ correct: boolean, correctOptionId, explanation }` — this is the ONLY place correctOptionId is revealed (per UX-02, D-12)

    f) **GET /:id/quiz/practice** — Get practice quiz questions for completed lessons
       - Fetch all quiz questions from completed lessons of this roadmap
       - Return randomized subset (up to 10 questions)
       - Same security rule: strip correctOptionId and explanation from response

    **3. Create `worker/src/routes/qa.ts`** — Hono route group for RAG Q&A:

    Export a Hono instance `qaRoutes` with these endpoints (all require authGuard + sanitize):

    a) **POST /ask** — Ask a question (RAG-backed)
       - Accept `{ question, roadmapId, lessonId? }` in body
       - Verify roadmapId belongs to current user (IDOR prevention, QNA-03)
       - Embed question: `env.AI.run("@cf/baai/bge-large-en-v1.5", { text: [question] })`
       - Query Vectorize with metadata filter: `{ roadmapId, userId }` (plus `lessonId` if provided for in-lesson scope)
       - TopK: 5 results, returnMetadata: "all"
       - Fetch lesson titles from D1 for citations
       - Build context string from chunk metadata
       - Generate answer: `env.AI.run("@cf/meta/llama-3.3-70b-instruct-fp8-fast", { messages: [system_with_context, user_question] })`
       - System prompt MUST include: "Answer based only on the provided context. Reference lessons by their title. If the context doesn't contain relevant information, say so."
       - Return `{ answer, sources: [{ lessonId, title, displayText }] }` (per QNA-04)

    **Security on ALL routes:**
    - Every route group applies `authGuard` middleware (sets userId from session)
    - POST/PUT/PATCH routes apply `sanitize` middleware
    - Every query on user-owned data includes userId filter (IDOR prevention)
    - Rate limiting: add `rateLimitMiddleware` on AI-calling endpoints (chat message, qa ask)
