    **1. Create `apps/web/app/components/qa/QAThread.tsx`** — reusable Q&A conversation thread:

    Props:
    ```typescript
    interface QAThreadProps {
      roadmapId: string;
      lessonId?: string; // If provided, scopes to this lesson (QNA-01); if null, full roadmap (QNA-02)
      placeholder: string; // "Ask about this lesson..." or "Ask about this roadmap..."
      emptyText: string; // Empty state text
    }
    ```

    **Layout:**
    - Full-height scrollable message area (use shadcn ScrollArea) + fixed bottom input bar
    - Same bubble style as buddy chat (per UI-SPEC — AI left-aligned bg-card, user right-aligned bg-foreground/8)
    - Auto-scroll to bottom on new message

    **Input bar:**
    - shadcn `Input` component with passed `placeholder` prop
    - Send button: `Button` icon-only with `Send` (Lucide), 48px minimum
    - Send on Enter, disabled when empty or request in flight

    **Q&A flow:**
    - On send: call `askQuestion(question, roadmapId, lessonId)`
    - Show typing indicator (3 animated dots) while waiting
    - Render AI response with inline citation links

    **Citation rendering (D-15, QNA-04):**
    - Parse the answer text for citation patterns: `[Lesson N: Title]`
    - Also use the `sources` array from the API response
    - Render citations as inline `<a>` elements:
      - Label role (14px), accent color (`text-primary`), underline on hover
      - Click navigates to `/roadmaps/${roadmapId}/lessons/${source.lessonId}`
      - If inside a bottom sheet: close the sheet first, then navigate
    - Citations are NOT external links — all internal app routes

    **Empty state:**
    - Centered: `emptyText` prop (Body, 16px, muted-foreground)
    - No CTA — just the input bar

    **Error state (per UI-SPEC):**
    - AI bubble with `border-destructive`: "Couldn't retrieve an answer. Try rephrasing your question."

    **2. Create `apps/web/app/components/qa/InLessonQASheet.tsx`** — bottom sheet wrapper per D-17, UI-SPEC Screen 4:

    Props:
    ```typescript
    interface InLessonQASheetProps {
      open: boolean;
      onOpenChange: (open: boolean) => void;
      roadmapId: string;
      lessonId: string;
    }
    ```

    **Implementation:**
    - Use shadcn `Sheet` component (side="bottom")
    - Height: 60% of viewport by default (use CSS: `h-[60vh]`)
    - `role="dialog"`, `aria-label="Lesson Q&A"`, `aria-modal="true"` (per UI-SPEC accessibility)
    - Focus trap inside sheet; first focusable element (input) receives focus on open
    - On close: focus returns to the "Ask AI" button (the trigger)
    - Background scrim: 50% opacity (Sheet component handles this by default)
    - Animations: slide-up 200ms ease-out on open, slide-down 150ms ease-in on close (Sheet handles animations)

    **Content:** Renders `QAThread` with:
    - `roadmapId` and `lessonId` passed through
    - `placeholder`: "Ask about this lesson..." (per UI-SPEC copywriting)
    - `emptyText`: contextual empty text

    **Citation click behavior:**
    - When a citation is clicked inside the sheet: close the sheet first, then navigate to the cited lesson
    - This is handled by passing an `onCitationClick` callback to QAThread that calls `onOpenChange(false)` then navigates
