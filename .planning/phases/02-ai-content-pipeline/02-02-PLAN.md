    Create `worker/src/workflows/ContentGenerationWorkflow.ts` implementing a Cloudflare Workflow with 4 steps.

    **Import pattern:**
    ```typescript
    import { WorkflowEntrypoint, WorkflowEvent, WorkflowStep } from "cloudflare:workers";
    import { drizzle } from "drizzle-orm/d1";
    import { eq } from "drizzle-orm";
    import * as schema from "../db/schema";
    import { RoadmapOutputSchema, LessonOutputSchema, QuizOutputSchema } from "../validation/content-schemas";
    import { buildRoadmapSystemPrompt, buildLessonSystemPrompt, buildQuizSystemPrompt, ROADMAP_JSON_SCHEMA, LESSON_JSON_SCHEMA, QUIZ_JSON_SCHEMA } from "../validation/roadmap-prompts";
    import { chunkText } from "../services/chunk-text";
    ```

    **Payload type:**
    ```typescript
    type ContentPayload = {
      topic: string;
      userId: string;
      conversationId: string;
    };
    ```

    **Step 1: "generate-roadmap"** (retries: 3, delay: 5s, backoff: exponential)
    - Call `this.env.AI.run("@cf/meta/llama-3.3-70b-instruct-fp8-fast", { messages: [...], response_format: { type: "json_schema", json_schema: ROADMAP_JSON_SCHEMA } })`
    - Parse the response text as JSON, then validate with `RoadmapOutputSchema.parse()`
    - Insert into `roadmaps` table with: id = `crypto.randomUUID()`, status = "generating", nodesJson = `JSON.stringify(validated.nodes)`
    - Return ONLY the roadmapId (string) — stays under 1MiB limit (per STATE.md decision)

    **Step 2: "generate-lessons"** (retries: 2, delay: 10s)
    - Fetch roadmap from D1 by roadmapId
    - Parse nodesJson to get node array
    - For each node: call AI with buildLessonSystemPrompt(), validate with LessonOutputSchema, insert into `lessons` table
    - Use deterministic IDs: `${roadmapId}-lesson-${node.id}` for idempotency on retry
    - Return array of lessonIds (string[]) — small, well under 1MiB

    **Step 3: "generate-quizzes"** (retries: 2, delay: 10s)
    - For each lessonId from step 2: fetch lesson content from D1
    - Call AI with buildQuizSystemPrompt(lesson.content), validate with QuizOutputSchema
    - Insert quiz row, then insert quiz_questions rows with deterministic IDs
    - Each question: store optionsJson as JSON string, store correctOptionId and explanation
    - Return nothing (void step — all data written to D1)

    **Step 4: "embed-content"** (retries: 3, delay: 15s)
    - For each lessonId: fetch lesson from D1, chunk with `chunkText(lesson.content, 300, 50)`
    - For each chunk: call `this.env.AI.run("@cf/baai/bge-large-en-v1.5", { text: [chunk] })`
    - Upsert to Vectorize: `this.env.VECTORIZE.upsert([{ id: "${lessonId}-chunk-${i}", values: embedding.data[0], metadata: { lessonId, roadmapId, userId, chunkIndex: i, text: chunk, lessonTitle: lesson.title } }])`
    - After all embeddings: update roadmap status to "complete" in D1

    **Error handling:**
    - Wrap the entire `run()` in try/catch
    - On any unhandled error: update roadmap status to "failed" in D1
    - Each step.do() already has retries configured — let Workflows handle step-level retries

    **CRITICAL CONSTRAINTS from Research:**
    - NEVER combine `response_format` and `stream: true` (Pitfall 2)
    - NEVER return full content from step.do() — only IDs (1MiB limit, Pitfall 1)
    - Chunk at 300 words max (Pitfall 3 — bge-large 512-token limit)
    - Use deterministic IDs for idempotency on retry (Pitfall — partial failure recovery)

    Create `worker/src/services/content-generation.service.ts` with helper functions used by the chat API route (Plan 03):

    **1. `detectRoadmapIntent(message: string): boolean`**
    - Keyword heuristic for MVP (per Research Pitfall 4 recommendation — confirmation step in chat, not an AI classifier call)
    - Match patterns indicating roadmap creation intent:
      - "create a roadmap" / "make a roadmap" / "build a roadmap" / "generate a roadmap"
      - "teach me about" / "I want to learn" / "help me learn"
      - "create a learning path" / "learning plan for"
    - Use case-insensitive regex matching
    - Return `true` if any pattern matches
    - This is deliberately simple — the chat API will add a confirmation step before triggering the Workflow

    **2. `buildChatMessages(history: ChatMessage[], newMessage: string): AiMessage[]`**
    - Takes the last 20 messages from D1 and the new user message
    - Returns array formatted for Workers AI: `[{ role: "system", content: BUDDY_SYSTEM_PROMPT }, ...history, { role: "user", content: newMessage }]`
    - BUDDY_SYSTEM_PROMPT: "You are Mimir, a friendly and encouraging learning companion. You help users explore topics, answer questions about learning, and suggest when a learning roadmap might be helpful. Keep responses concise and conversational. When a user wants to learn a specific topic in depth, suggest creating a roadmap."
    - Limit history to last 20 messages to stay within Llama 3.3 24,000-token context window (per Research recommendation)

    **3. `extractTopicFromMessage(message: string): string`**
    - Extracts the topic from a roadmap intent message
    - Strips common prefixes: "create a roadmap for", "teach me about", "I want to learn", etc.
    - Returns cleaned topic string
    - Used when triggering the Workflow to pass a clean topic, not the full user message

    **Types:**
    ```typescript
    export interface ChatMessage {
      role: "user" | "assistant";
      content: string;
    }

      content: string;
    }
    ```
